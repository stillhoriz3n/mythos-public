How it works

Two new functions inside substrate (plpython3u on joey-mac, port 5435):

substrate.plm_derive(question) — Tier 0
1
Parse intent. Is this a listing question ("what are the...") or an explanation question ("why does...")? This determines whether substrate can answer alone.
2
Try 6 constitutional routes: invariants, census, identity, forces, compositions, keyword search. Each route is a SQL query against live blobs.
3
If a route matches: return structured JSON with the answer + which blobs it derived from. Cost: 0. Latency: ~2ms.
4
If no route matches: return escalate: true + any relevant context_blobs found during the search. This context gets packed into the model's system prompt.
substrate.ask_self(question) — Full Pipeline
1
Call plm_derive(). If it answers → return immediately. Done.
2
If escalated → gather context: the blobs plm_derive found + 5 core invariants.
3
Build system prompt: Isaac's identity + Ten Laws + substrate context.
4
POST to 10.69.10.121:8080 (Qwen3-14B on 7900XT via WireGuard mesh). Isaac responds, grounded in constitutional context.
5
Return Isaac's response, prefixed with tier + sources.

The 6 routes

What plm_derive() checks, in order:

Constitutional Routes (Tier 0)
1
Invariants. 120 constitutional claims. "What are the rules?" → returns them all, derived from doctrine/invariant_* blobs.
2
Census. "How many blobs?" → 48,729 blobs, 163 kinds, 46 compositions. Live count from substrate.blob.
3
Identity. "Who is JoeA?" → looks up principal/alpha_oa/joea and returns the blob's fields including justification.
4
Forces. "What can the substrate do?" → queries substrate.force_registry and lists all registered forces.
5
Compositions. "What kinds exist?" → returns all 56 composition_definition blobs with their tiers.
6
Keyword search. Fallback — ILIKE against blob names, descriptions, and claims. Filters out noise (ingested_document, insert kinds). Returns top 10 matches.

If the question asks for explanation (detected via "why", "explain", "how does"), it always escalates to Tier 1 — but still gathers context blobs during the search to give the model grounding.
