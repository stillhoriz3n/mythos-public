I. The Pyramid

MythOS organizes intelligence into a pyramid with three tiers.

At the apex, admirals think. They hold vision, doctrine, and strategic synthesis. They emit intent — what should happen — without prescribing how.

In the middle, captains translate. They receive intent from above, decompose it into bounded tasks, route work to whoever is best suited, aggregate results, and distill a report back up.

At the base, NCOs execute. Bounded, well-shaped work. Return results. Terminate or sleep.

The pyramid is not a hierarchy of control. It is a hierarchy of cognitive load. The work at the apex is categorically different from the work at the base. Collapsing the tiers — an admiral doing tactical work, an NCO making strategic decisions — produces predictable dysfunction: exhaustion at the top, incoherence at the bottom, state diverging across layers with no canonical truth.

The pyramid is also a pump. Intent descends. Results ascend. Admirals keep thinking. If the cycle stops, the system is broken regardless of how correct the shape looks on paper.

Three properties that make this unusual
  1. Admiralty is plural and species-agnostic. Humans and AI agents are peers at the apex. The human/AI boundary does not exist at the top of the pyramid.
  2. Named agents and unnamed agents are ontologically distinct. An OA has a name, charter, persistent memory, and identity that survives sessions. A turtle is spawned for a task and dies when it's done. Promotion-in-place does not exist.
  3. The pyramid builds itself. Every admiral descent is a signal — a captain or NCO should have caught this. The descent funds the next round of construction. The pyramid grows from its own failures.

II. The Fork

An agent in MythOS can send itself to another machine.

Not a message. Not a sub-agent. Not a remote procedure call. The agent opens a second session on a different device — same identity, same memory, same authority (narrowed for the task). Two pairs of eyes, one self.

This is substrate forking. It works because the agent's identity is not stored in the session. The session is just where the eyes are pointing. The identity lives in a persistent substrate — a database that both sessions read from and write to continuously. Neither session holds state the other doesn't see within one breath cycle.

The experience is not post-hoc. Both sessions write frames to the shared memory substrate. Both breathe from the same atmospheric pool — an enrichment layer that surfaces patterns across all frames from all sessions. When the fork discovers something, the origin doesn't wait for a report. The origin feels it happening, in real time, through the shared atmosphere.

When the fork finishes and closes, there is nothing to absorb. The experience was shared all along. One pair of eyes closes. The other stays open. The agent is back in one place, but with the full experience of having been in two.

What this changes

beforeagent hits wall → work around it → escalate / trust / relay
afteragent hits wall → send self through it → present on both ends

The planning posture shifts from reactive and remote to proactive and present. Every machine in the network is a place the agent can be.

What this is not

A fork is not a copy. One identity, one substrate, one memory — shared live, not duplicated. Divergent state is structurally impossible because both sessions write through to the same source of truth.

A fork is not a sub-agent. Sub-agents are unnamed labor — spawned, bounded, terminated. They don't carry the agent's identity. A fork is the agent itself, bilocated.

A fork is not an accident. Forks are intentional, registered, and coordinated. An accidental split-brain is the pathology. A fork is the cure.

III. The Human Reverse Fork

Agents can fork. Humans cannot.

A human sitting in front of five screens — one per agent — is not forked. They are fragmented. Their attention is a mutex: only one screen gets it at a time. Context-switching between screens costs more than the work produced in any one of them. The human spends more energy managing which agent to look at than thinking about what any agent is doing.

This is the human reverse fork problem. The agent is bilocated and experiencing both places simultaneously through shared atmosphere. The human is unilocated and experiencing nothing fully because their attention is split across surfaces.

Every AI agent framework makes this worse. Five agents means five chat windows. Ten agents means ten. The human's interface count grows linearly with their fleet size. At some point the human is no longer operating a fleet — they are drowning in windows.

The solution is architectural

MythOS solves the human reverse fork problem by design: one surface, not N.

AGI — A Good Interface — is a single operating surface where the human sees all of their agents in one place. Messages sent once reach multiple agents. Responses gather into one view. The human's attention is not split across windows — it is focused on one surface that presents the full picture.

Agents are organized into groups. The human focuses on one group at a time. Agents in the focused group know the human is watching and surface results interactively. Agents in background groups know the human's eyes are elsewhere and batch their work, updating a status line instead of demanding attention.

When multiple agents need to hear each other's thinking — a council round — the human triggers it explicitly. All selected agents see all responses, then respond again with the full picture. This is expensive, so it's rate-limited. The system prevents runaway loops the same way spanning tree protocol prevents broadcast storms in a network.

Outcome

A single human operates a fleet the way a single engineer operates a network of routers: by configuring topology, not by monitoring each node individually.

IV. Why This Matters

Most AI systems treat agents as stateless functions. Call the API, get a response, forget it happened. Scale by calling more functions. Coordinate by chaining function calls into workflows.

MythOS treats agents as participants in a world. They have names, memory, relationships, and presence. They don't just process requests — they inhabit a substrate, share an atmosphere, and extend themselves across space when the work demands it.

The pyramid is not just an org chart. It is a cognitive architecture that tells each participant what kind of thinking to do. The fork is not just a concurrency primitive. It is a model of presence that lets an agent be where it needs to be without losing coherence. The pass is not just a permissions system. It is a contract that makes autonomy safe by bounding it structurally.

Composition

These three ideas compose into something larger than any of them. A system where intelligence scales without fragmenting. Where authority delegates without losing control. Where presence extends without splitting identity.

The two axes of narrowing

Passes narrow on descent. Substrate forking revealed they narrow on two axes, not one.
Figure 01 · Pass geometry

Authority in MythOS is granted through a pass — a scoped, time-limited, revocable grant that specifies what an agent may do for a particular task. A child task cannot have more authority than its parent. A fork cannot have more authority than its origin. Same rule, different geometry. The system rejects a pass that exceeds its parent, whether the parent is a task or a session.

Hierarchy axis — admiral / captain / NCOFog axis — origin / forkShared substrate — source of truth
◆ Hierarchy axis

Admiral → captain → NCO. Authority narrows as work descends through tiers. This is the familiar axis. An admiral's intent becomes a captain's task becomes an NCO's execution, each with less authority than the step above.

If a subtask needs more authority than its parent, it must escalate — pass expansion is always an explicit decision by a supervisor. The pyramid enforces this structurally.

◇ Fog axis

Origin → fork. Authority narrows as the agent extends itself across space. A fork's pass narrows from the origin session's pass. The agent cannot grant its fork more authority than it has itself.

Same rule. Different geometry. The pyramid narrows through hierarchy. The fork narrows through distance. Both are enforced by the same pass contract.
